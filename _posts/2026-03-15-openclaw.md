随着 AI Agent 的快速发展，我开始思考一个问题：能不能让 AI 真正"住"在我的电脑里，帮我执行任务，而不只是回答问题？

OpenClaw 就是我找到的答案。它是一个运行在本地的 AI Agent，通过飞书、Telegram、微信等你已经在用的消息平台接受指令，然后在你的电脑上执行真实的操作——改文件、跑脚本、抓网页、发邮件。不在电脑旁边的时候，拿起手机发一条消息，任务就开始了。

本文记录我安装和使用 OpenClaw 的过程，以及它目前在我日常工作和生活中的具体应用。

---

## 一、我为什么安装 OpenClaw

有几个痛点一直困扰我：

**不常带电脑，但有时需要紧急改代码**。以前在外面遇到线上问题，要么赶回家，要么拜托同事，效率很低。

**每天的新闻获取太碎片化**。之前靠微信推送和网易新闻，信息质量参差不齐，还要手动去找。

**学习计划执行不稳定**。英语单词背诵这类事情，计划定了很多遍，真正坚持下来的少。

**想把一些重复的开发任务自动化**。比如把架构梳理好之后，让 AI 帮我生成页面框架代码，不用每次从头写。

这几个需求加在一起，让我决定试试 OpenClaw。

---

## 二、安装与基础配置

OpenClaw 基于 Node.js，在 Mac 上安装相对简单：

```bash
# 安装 OpenClaw
npm install -g openclaw

# 初始化 Gateway（控制中枢）
openclaw gateway install

# 启动
openclaw gateway start
```

安装完成后，通过菜单栏的 OpenClaw 图标可以看到 Gateway 的运行状态。

**连接飞书**

我选择飞书作为主要的指令通道，因为工作中本来就在用。在 OpenClaw 的配置中添加飞书机器人的 webhook 和 token，配置完成后在飞书给机器人发一条消息，OpenClaw 就能收到并响应。

**权限配置**

安装时最重要的一步是设置权限规则。我的配置原则是：

- 删除文件：必须向我确认，不允许静默执行
- 访问隐私目录（如 `~/Documents`）：需要我授权
- 发送文件到外部：需要我确认
- 执行 Git 操作：允许，但需要我确认 commit message

这样配置之后，OpenClaw 有足够的操作权限，但关键操作都有人工确认，不用太担心误操作。

## 三、我的实际使用场景

### 3.1 远程改代码、推 Git

这是我用得最多的功能。不带电脑的时候，如果有紧急的 Bug 修复，直接在飞书发：

> 帮我打开 project-x/src/utils/request.ts，把第 42 行的超时时间从 5000 改成 10000，然后 commit 并 push 到 main 分支。

OpenClaw 会打开文件、定位到对应行、做修改、执行 `git commit` 和 `git push`，完成后在飞书回复我结果。整个过程不需要我回家，也不需要拜托别人。

在这之前，遇到这种情况只能赶回去或者远程连接，效率很低。现在直接在手机上操作，省了大量来回折腾的时间。

### 3.2 每日新闻摘要

我设置了一个每天早上 8 点执行的定时任务：

```
每天 08:00 抓取以下来源的今日要闻：
- 36氪
- 少数派
- Hacker News Top 10
整理成简报发送到飞书
```

现在每天早上打开飞书，就能看到一份整理好的新闻摘要，不需要再手动去各个平台翻。信息密度比以前靠微信推送高很多，而且是我自己定义的来源，质量更可控。

### 3.3 英语单词定时任务

背单词这件事我定了很多次计划，执行率一直不高。现在用 OpenClaw 做了一个定时提醒：

每天 21:00，飞书自动给我推送当天的 20 个单词，每个单词附带例句。我只要在手机上看一遍，花不了几分钟。坚持这个习惯比以前靠意志力容易多了，因为提醒本身就是自动的。

### 3.4 辅助代码生成

我通常会先把模块的架构、数据结构、接口定义整理成文档，然后发给 OpenClaw：

> 根据 docs/user-module-spec.md 的接口定义，生成 Android 端的 UserRepository 类，使用 Kotlin，包含 getUser、updateUser、deleteUser 三个方法，错误处理用 Result 包装。

OpenClaw 会读取文档，生成符合规范的代码，直接写入指定文件。不是最终可用的代码，但框架和样板代码都有了，剩下的填充业务逻辑快很多。

### 3.5 生成课件和学习动画页面
这个用法是我意外发现的收获之一。
学习新技术的时候，我会把想理解的概念发给 OpenClaw，让它生成一个可以直接在浏览器打开的 HTML 学习页面：

帮我生成一个讲解 Transformer 注意力机制的 HTML 页面，要有分步骤的动画演示，用简单的色块和箭头展示 Query、Key、Value 的计算过程，保存到 ~/Documents/learning/transformer-attention.html

OpenClaw 会生成一个带 CSS 动画的单页 HTML 文件，直接双击就能在浏览器里看，不需要任何额外环境。动画可以一步一步点，配合自己的节奏来理解。
对于一些需要反复看的知识点，我会让它做成「课件」形式：

把 Kotlin 协程的生命周期整理成一个带翻页效果的 HTML 课件，每页一个知识点，配上代码示例，保存到 ~/Documents/learning/kotlin-coroutine.html

这种方式比看视频更灵活——可以随时暂停、跳页、在浏览器里直接修改内容做笔记。以前学习主要靠视频和文档，现在多了一种交互式的学习材料，理解复杂概念的效率明显提高了。

## 四、安全问题

装之前我也犹豫过——电脑上有很多私人文件和代码，给一个 AI 这么大的权限，安全吗？

最终我的判断是：权限规则配置好的前提下，方便性大于风险。

我的具体做法：
- 删除、发送、访问隐私目录等高风险操作全部设为需要确认
- 不安装不常用的 Skill（OpenClaw 的功能扩展），减少攻击面
- 飞书 bot 的 token 只配置在本地，不同步到任何云端

值得注意的是，OpenClaw 的 ClawHub（技能市场）存在一定风险——有报道指出第三方 Skill 中出现过恶意代码。建议只安装官方维护的 Skill，不轻易安装来源不明的第三方扩展。

---

## 五、耗电问题

用了一段时间后，发现 MacBook Pro 的续航明显变短了。即使盖上盖子，电量下降的速度也比以前快。

原因是 OpenClaw 的 Gateway 进程长期在后台运行，即使在待机状态下也有一定的 CPU 和网络活动。

目前的解决方案：
- 不需要远程控制的时候，从菜单栏暂停 Gateway
- 考虑把 OpenClaw 部署到家里的 Mac Mini 上，让 MacBook 只作为指令发送端，不承担计算负载

如果你主要在固定地点使用，建议用 Mac Mini 这类不需要考虑电池的设备来跑 OpenClaw，效果会更好。

---

## 总结

OpenClaw 改变了我和电脑之间的关系——以前必须坐在电脑前才能做的事，现在拿起手机就能完成。对于习惯多端工作、经常不在电脑旁边的开发者来说，它带来的效率提升是实实在在的。

目前对我价值最大的三个场景是：远程 Git 操作、每日新闻摘要、英语单词定时推送。后续打算继续探索更多自动化工作流，比如接入日历管理和项目进度跟踪。

如果你也有"不在电脑前但需要操作电脑"的痛点，可以考虑试试。权限配置花点时间做仔细，用起来安全感会高很多。
