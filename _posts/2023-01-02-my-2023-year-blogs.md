## 关于工作

2022 年的工作重心依然是聚合配送平台的移动端开发，Android 和 iOS 双端并行。

这一年国家出台了多项 App 合规政策，基本每隔一个季度就有新的要求落地，大量时间耗在了政策合规修改和历史遗留 Bug 的修复上。技术上没有太多突破，更多是维护性的工作——公司对马甲包和平台分发的处理，写了一些自动化打包脚本来提升效率。

开发节奏是先写完 Android，再复用接口和业务逻辑重新梳理写 iOS。因为逻辑已经跑通一遍，iOS 端的开发速度明显更快。公司早期技术选型全是原生开发，Flutter 和 RN 在项目里几乎没有用武之地，日常工作以业务逻辑开发为主，相对基础。

这种状态让我意识到，单纯的业务开发很难带来技术成长，必须主动在工作之外找方向。

## 关于学习

**Golang 的入门**

公司的后端是 PHP 架构，高并发场景下表现一般。我开始调研有没有更合适的语言替代方案，最终选择了 Golang。

Go 在并发处理上的确优势明显，goroutine 的轻量设计比 Java 线程更灵活，虽然 Java 的 Netty 在 NIO 上也有不错的表现，但整体上手体验还是 Go 更简洁。

有之前多语言学习的积累，Go 上手速度比预期快。不过学下来也发现了一些不足：Go 的生态和工具链远不如 Java 完善，很多 Java 里一个注解就能搞定的事，Go 需要手写大量样板代码来支撑。这是取舍，并发性能换来的是生态上的妥协。

多学一门语言，收获的不只是语法，更是一种新的思维方式。

**架构思维的系统学习**

这一年开始有意识地从"业务开发者"向"架构设计者"转变，买了几本架构相关的书系统阅读，开始思考系统设计的全局视角，而不只是单点功能的实现。这个转变比学一门新语言更难，但也更有价值。

**人工智能与大数据的持续跟进**

AI 和大数据的学习没有停，继续保持推进节奏。

**K8s 的复习**

工作中没有机会用到 K8s，只能靠自己搭环境练习来保持手感。知识如果不用，会比想象中忘得更快，碎片化学习虽然效率不高，但总比完全搁置要好。

## 关于生活

2022 年是疫情管控最严的一年，核酸几乎成了生活的一部分，每周一次，有一段时间因为形势紧张改为居家办公。活动范围很小，生活轨迹基本就是家和公司两点一线。

年底管控逐渐放开，生活才慢慢恢复了一点烟火气。

这一年没有太多值得记录的生活事件，但技术和学习的积累是扎实的。有时候，平静的外表下其实藏着很多东西。
