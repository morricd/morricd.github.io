OC 与 Swift 混编是 iOS 项目迁移过程中的常见需求，两者可以在同一项目中相互调用，但需要通过桥接文件来打通。混编有两个方向：在 Swift 项目中引入 OC 代码，以及在 OC 项目中引入 Swift 代码。

> **混编的核心限制**：OC 自定义类**不能继承自 Swift 类**，即需要与 Swift 混编的 OC 类不能成为 Swift 类的子类。反方向则没有限制——Swift 类可以继承自 OC 类。
{: .prompt-warning }

---

## 一、在 Swift 项目中混编 OC

### 1.1 创建桥接文件

在 Swift 项目中新建 OC 文件时，Xcode 会弹出提示，询问是否自动创建桥接文件（Bridging Header）。选择**创建**，Xcode 会自动生成 `项目名-Bridging-Header.h` 并在 Build Settings 中配置好路径。

![创建 OC 文件时提示生成桥接文件](/assets/images/2022/2022031801.png)

![确认生成桥接文件](/assets/images/2022/2022031802.png)

![自动生成的桥接文件](/assets/images/2022/2022031803.png)

在 Build Settings 中可以看到桥接文件的路径配置：

![Build Settings 中的桥接文件路径](/assets/images/2022/2022031804.png)

### 1.2 Swift 调用 OC

在自动生成的桥接文件（如 `SwiftAndOC-Bridging-Header.h`）中，导入需要在 Swift 中使用的 OC 头文件：

```objc
#import "MyOCClass.h"
```

导入后，Swift 文件中可以直接使用该 OC 类，无需额外操作。

![在桥接文件中导入 OC 头文件](/assets/images/2022/2022031805.png)

### 1.3 OC 调用 Swift

项目编译时，Xcode 会自动生成一个 Swift 暴露给 OC 的头文件，命名规则为：`项目名-Swift.h`。OC 文件中导入该文件即可调用 Swift 代码。

![自动生成的 Swift 桥接头文件](/assets/images/2022/2022031806.png)

### 1.4 Swift 类的暴露规则

Swift 类若要被 OC 调用，必须满足以下条件：

1. **继承自 `NSObject`**：只有继承了 `NSObject` 的 Swift 类才能被 OC 访问；
2. **添加 `@objc` 标注**：需要在 OC 中可见的属性和方法，必须加上 `@objc` 修饰，否则在 OC 中不可见。

```swift
import Foundation

@objc class MySwiftClass: NSObject {

    @objc var name: String = ""

    @objc func sayHello() {
        print("Hello from Swift")
    }

    // 未加 @objc，OC 中不可见
    func swiftOnlyMethod() {
        print("Only for Swift")
    }
}
```

![Swift 类继承 NSObject 并添加 @objc](/assets/images/2022/2022031807.png)

### 1.5 在 OC 文件中导入 Swift 桥接头文件

```objc
#import "SwiftAndOC-Swift.h"

// 即可使用 Swift 类
MySwiftClass *obj = [[MySwiftClass alloc] init];
[obj sayHello];
```

![在 OC 中导入自动生成的 Swift 头文件](/assets/images/2022/2022031808.png)

## 二、在 OC 项目中混编 Swift

在 OC 项目中新建 Swift 文件时，Xcode 同样会提示是否生成桥接文件，点击**创建**即可，路径配置与上文一致。

两个方向的调用规则如下：

**Swift 调用 OC**：在桥接文件中导入需要使用的 OC 头文件：

```objc
// 项目名-Bridging-Header.h
#import "MyOCClass.h"
```

**OC 调用 Swift**：在 OC 文件中导入自动生成的 `项目名-Swift.h`：

```objc
#import "MyProject-Swift.h"
```

---

## 总结

| 场景              | 操作                                                 |
| ----------------- | ---------------------------------------------------- |
| Swift 调用 OC     | 在 `项目名-Bridging-Header.h` 中 `#import` OC 头文件 |
| OC 调用 Swift     | 在 OC 文件中 `#import "项目名-Swift.h"`              |
| Swift 类暴露给 OC | 继承 `NSObject`，属性和方法加 `@objc`                |
| OC 继承 Swift 类  | ❌ 不支持                                             |
| Swift 继承 OC 类  | ✅ 支持                                               |
