> 为了工作，更为了生活。

> 技术持续演进，学习多种语言是适应变化的必然选择。不同技术之间往往有诸多共通之处，旧的被淘汰，新的又涌现——归根结底，正是工作需求的不断变化，驱动着我们持续学习与成长。

## 技术更迭与语言淘汰

Flash 的落幕，是很多那个年代开发者共同的记忆。苹果率先宣布不再支持，Adobe 随后正式停止维护，一门曾经风靡全球的语言就此谢幕。那段时间我确实迷茫过，但很快就想清楚了：技术会过时，但编程这条路值得走下去。

于是我把目光转向了当时正处于上升期的 iOS 开发，同时自学 PHP 作为后端方向。自学两个多月，做出几个 demo，便开始找工作。那时 iOS 开发人才稀缺，我顺利拿到了第一份工作。

进入公司后，因为团队人员流动，我逐渐接手了 Android 和 ASP 的开发维护工作。学习 Android 的过程中，我惊喜地发现它与 iOS 在 UI 架构模式上有不少相通之处——这让上手速度快了很多，利用晚上的时间就基本掌握，并顺利接手了项目维护。ASP 部分则因为大学打过基础，维护起来相对从容。

## 创业带来的全面成长

互联网高速发展的大背景下，我萌生了创业的念头，于是成立了自己的技术公司，业务涵盖网站建设、App 开发和微信公众号平台等。为了提升开发效率，我们基于开源 PHP 框架对公众号平台进行二次开发。这一阶段，也是我真正完成"前端到后端"全栈跨越的时期。

随着业务拓展，部分客户提出了用 Java 开发的需求。得益于之前的 .NET 和数据库基础，学习 Java 并没有太陡峭的曲线，很快便应用到了实际项目中。

创业持续了两年，最终因为对市场理解不够深入、团队管理经验欠缺，我选择了关闭公司，进入下一个阶段。虽然结果并不圆满，但这段经历让我在技术、管理、商业判断等多个维度都得到了难以复制的历练。

## 管理岗位与技术深耕

创业结束后，我先后进入几家公司担任技术管理岗位。这些角色不仅要求掌握前沿技术，更要具备系统性的架构思维。后端技术栈也随着职责扩大而持续演进：从 Java 单体架构，到 SOA 服务化，再到微服务架构，一步步完成了技术认知的升级。

为了满足业务多样化的需要，我陆续学习并应用了 Golang，参与了实际项目开发；随着数据业务的成长，又深入学习了 Hadoop、Hive、Flink 等大数据技术；面对 AI 浪潮的涌来，也开始系统学习人工智能相关知识与工程实践。

移动端方面，随着原生开发语言的迭代，Kotlin 与 Swift 也先后纳入了我的技术栈。

即便在管理岗位，我始终没有放弃在技术上的持续突破。这些年走过来，我越来越觉得：**保持学习，是应对一切变化最可靠的底气。**
